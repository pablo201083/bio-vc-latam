# Startup Profiles Minimum Viable

Esta capa busca crear una `unidad minima de verdad` por startup para sostener:

- taxonomia bottom-up
- decisiones de `include / review / exclude`
- visualizaciones
- cola editorial
- investigacion futura

## Principio

Cada startup debe tener, como minimo:

1. una descripcion corta comparable
2. una fuente concreta
3. una explicacion de por que entra o no entra en tesis
4. una senal minima de problema, enfoque y mercado

## Campos clave

- `startup_summary_v1`
- `business_one_liner`
- `problem_addressed`
- `technical_approach`
- `industry_destination`
- `materiality_signal`
- `thesis_scope_note`
- `source_url`
- `source_type`
- `source_date`
- `evidence_excerpt`
- `summary_confidence`
- `review_status`

## Regla editorial

No copiar slogans tal cual.

Preferir:

- texto oficial parafraseado
- lenguaje simple
- una frase sobre que hace
- una frase sobre por que importa para el universo thesis-first

## Orden de trabajo recomendado

1. startups `review`
2. startups `include` con score de calidad bajo
3. hubs o startups con mas centralidad
4. resto del universo

## Estados sugeridos

- `seeded`
- `needs_research`
- `reviewed`
- `verified`
